- PowerPoint roadmap labels need bounded text boxes. When a bar reaches the right edge, do not push labels into a tiny outside column; keep labels inside the bar or give them enough measured room, and test overlapping same-workstream initiatives.
- When a user asks for savings visualization and the app already distinguishes Savings from Avoidance, do not assume one metric is enough. Confirm whether the view needs Savings-only, Savings + Avoidance, or both, and test both paths with multi-pillar data.

- Treat the roadmap label and tooltip as separate surfaces: keep a short distinguishing label inside ranged bars, retain complete financial detail on hover, and render the PowerPoint output because shrink-to-fit behavior is not reliable enough by itself.
- When a business classification starts as a derived rule but the user later needs control, store an explicit normalized override and retain the old derivation only as a backward-compatible fallback.
- When a user asks for a rollup, aggregate the visual marks themselves at the requested level; grouping initiative-level marks under a shared heading is not a true rollup.
- Never use SVG `textLength` with `spacingAndGlyphs` to force a long executive label into a narrow bar. Select a shorter semantic label for the available width and retain the complete detail in the tooltip.
- A PowerPoint timeline bar is a timing mark, not a container that must hold every metric. Put approval totals and financial mix in stable large-type regions, and use an external callout for short spans instead of `fit: shrink`.
- When the user asks for PowerPoint to match an app view exactly, visual parity outranks a separate presentation interpretation. Reuse the same hierarchy, labels, colors, and responsive shortening rules rather than introducing a new narrative layout.
- Scenario snapshots must not own portfolio membership. Treat initiatives and their structure as a shared catalog, merge snapshots by stable ID, and test repeated switching from an already-stale scenario so newer initiatives cannot disappear.
- When an executive tracker combines overlapping dimensions such as approval, value type, and realized progress, convert them into mutually exclusive segments before drawing the bar. The visual must reconcile exactly to total portfolio value and make the goal gap the first-read story.
